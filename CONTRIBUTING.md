# Contributing to KDNA

Thanks for your interest in contributing. KDNA is an open ecosystem — the protocol, tools, domains, and Work Packs are built in the open.

## Quick Start

1. **Find something to work on** — Check open issues or propose a new one.
2. **Fork the repo** and create a branch.
3. **Make your change** — keep it focused. One PR = one logical change.
4. **Add tests** if applicable. CI must pass.
5. **Open a PR** with a clear description of what and why.

## Development Setup

```bash
# Clone and install
git clone https://github.com/aikdna/<repo>.git
cd <repo>
npm install        # Node.js repos
pip install -e .   # Python repos
swift build        # Swift repos
```

## Before Submitting

- [ ] Tests pass locally
- [ ] No new lint warnings
- [ ] Commit messages follow [conventional commits](https://www.conventionalcommits.org/)
- [ ] Documentation updated if needed

## Types of Contributions

- **Code**: bug fixes, features, performance improvements
- **Domains**: new `.kdna` judgment assets or improvements to existing ones
- **Work Packs**: new work capability packages or improvements
- **Documentation**: fixes, translations, tutorials
- **Issues**: bug reports, feature requests, RFC feedback
